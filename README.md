# 这个仓库的作用是什么
## 功能一

2025 年 7 月 26 日至 28 日，世界人工智能大会暨人工智能全球治理高级别会议（WAIC）将在上海世博中心和世博展览馆隆重举行。本届大会以 “智能时代 同球共济” 为主题，自 2018 年创办以来，已成功举办七届，始终秉持高端化、国际化、专业化、市场化、智能化的办会理念，现已成为全球人工智能领域极具影响力的行业盛会，是国家有关部门和上海市共同打造的国际高端合作交流平台。
OpenLoong 开源社区作为开源领域的积极推动者，一直致力于为开发者提供优质开源资源。值此大会之际，特面向全球开发者群体发起机器人作品征集活动。此次活动旨在充分展示开源技术在机器人领域的创新应用成果，促进社区技术交流，活跃社区氛围，吸引更多开发者投身开源项目，推动 OpenLoong 开源社区生态的繁荣发展。
活动设置丰富奖励，优秀项目将获得 2025 世界人工智能大会（WAIC）门票与专属展区曝光机会。这不仅为开源项目提供了更广阔的展示舞台，吸引全球目光，更是促进了开源技术与机器人产业的深度融合，激发更多创新灵感与合作机遇，助力推动整个行业的进步与发展。

## 功能二
### 作品提交要求
  - 提交资料
    - 将文本、代码、视频三个文件打包成一个文件夹，按规定命名，提交至指定 GitHub 仓库。文件夹命名格式为 “名字 - 联系方式（手机号） - 作品名称”，demo 及实物部署皆可。
    - 文件夹内容：
      1. 项目说明文档（word/PDF 格式）：文件命名为“[作品名称]- 项目说明文档.[docx/pdf]”。
        - 个人简介（50字左右）：明确要求参与者提供真实姓名、所在单位（学校或公司）、联系方式（手机号码、邮箱），并附上 50 字左右的个人简介（技术背景、开源项目经验或对开源生态的贡献与理解等）。
        - 作品介绍（150字左右）：作品介绍需涵盖项目名称、技术方向、应用场景等相关介绍，字数控制在 150 - 200 字，使评审人员能迅速把握作品亮点与价值。
      2. 技术文档（Word / Markdown 格式） ：文件命名为 “[作品名称]- OpenLoong 应用技术文档.[docx/md]”。
        - 详细阐述 OpenLoong 开源资料在作品中的应用情况，包括应用场景描述、技术原理分析、代码实现细节等内容
      3. 代码文件 ：提交完整可运行的代码，遵循良好规范，有清晰注释和合理目录结构。
      4. 视频文件（MP4 格式）：文件命名为 “[作品名称]- 演示视频.mp4”。
        - 代码及作品视频：作品视频应详细演示作品核心功能与操作流程，展示作品的实际运行效果。
        - 视频要求：视频清晰；横屏；视频文件大小不超过 500MB；食品时长不低于3min
      5. 其他辅助资料（非必须） *：命名为 “[作品名称]- 辅助资料.zip” 或 “[作品名称]- 辅助资料.rar”。
        - 如硬件原理图、3D 模型文件、设计图纸等，能够进一步辅助说明作品设计与实现过程的资料，统一放入一个压缩包。
    - 补充说明
      - 作品需使用 OpenLoong 开源资料，包括开源算法控制模块 / 核心框架等，作品说明文档要详细阐述其应用场景、技术原理及创新优化点。
      - 技术文档内容详实、逻辑清晰、排版规范，包含项目概述、系统架构、算法原理等关键部分，遵循统一模板。
      - 硬件形态不限，鼓励多样化创新，作品要展现特定场景下的独特优势与应用潜力，结合 OpenLoong 开源技术实现性能优化与功能拓展。
